: 요구사항 분석과 모델링 실시
	
요구사항 분석은
	시스템 외부에 존재하며 시스템에 이벤트를 주거나 정보를 받게 되는 사람 또는 시스템인 Actor를 식별하고, 
	추출된 기능적/비기능적 요구사항 중 
	기능적 요구사항으로 Actor가 특정 목표를 달성하기 위하여 
	해당 시스템에 수행해야 하는 일련의 행동 또는 독립적인 기능군인 Usecase를 식별해야 한다. 
	
요구사항 모델링은 
	식별된 Actor와 Usecase를 바탕으로 
	Actor와 Usecase간 상호작용 및 이벤트에집중하여 Usecase Diagram을 작성한다. 
	Usecase Diagram 작성 시 Actor와 Usecase, Usecase들 간의 관계를 명확히 하기 위해 
	Generalization, Association, Exclude, Include 관계를 사용하여 효과적으로 표현한다. 

마지막으로 각 Usecase에 대한 
	시나리오를 포함한 Usecase 명세서를 작성한다. 
	명세서에는 Usecase 개요, 관련 액터, 우선 순위, 선행 조건, 이벤트 흐름(시나리오), 후행 조건, 기타 요구사항으로 구성된다. 

이를 바탕으로 식당 주문 시스템에 대한 요구사항 분석 및 모델링을 아래와 같은 수행 흐름에 따라 실행한다.

타겟 시스템 : 식당 주문 시스템
Actor(시스템 외부의 사람 또는 시스템) : 고객, 사용자, 최고 시스템 관리자(admin)
Usecase :
	----- 고객, 사용자 -----
	ㅇ 로그인 : 고객, 사용자 -> 시스템에 로그인
	ㅇ 로그아웃 : 고객, 사용자 -> 시스템에 로그아웃
	----- 사용자 -----
	ㅇ 주문 메뉴 조회 : 사용자 -> 메뉴(이름, 가격, 설명) 조회
	ㅇ 추천 메뉴 조회 : 사용자 -> 고객이 추천한 메뉴를 조회
	ㅇ 주문 메뉴 구입 : 사용자 -> 메뉴 결제 <- 현금결제, 카드결제, 적립금 결제
	ㅇ 주문 내역 확인 : 사용자 -> 결제한 주문 내역(주문 일시, 주문 번호, 주문 내역) 확인
	ㅇ 언어 선택 : 사용자 -> 시스템의 사용언어(한, 영, 일, 중)을 변경
	ㅇ 매장 적립금 조회 : 사용자 -> 로그인 -> 매장 적립금 확인
	ㅇ 영수증 및 번호표 출력 : 사용자 -> 구매 내역에 대한 영수증 조회, 대기 번호표 출력
	
	ㅇ 문의하기 : 사용자 -> 시스템 -> 문의사항 (<- 고객)
	----- 고객 -----
	ㅇ 공지하기 : 직원(고객) -> 시스템 ->  공지사항( <- 사용자)
	ㅇ 메뉴 등록 : 직원(고객) -> 시스템 : 메뉴(이름, 가격, 설명)을 등록
		+ 일반 메뉴 등록
		+ 추천 메뉴 등록 : 직원(고객) -> 시스템 : 추천 메뉴 등록
	ㅇ 메뉴 수정 : 직원(고객) -> 시스템 -> 메뉴(이름, 가격, 설명)을 수정
		+ 일벤 메뉴 수정
		+ 추천 메뉴 수정
	ㅇ 매출 정산 : 직원(고객) -> 시스템 -> 일일 매출 정산 또는 월 매출 정산을 확인
		+ 일일 매출 정산 확인
		+ 월 매출 정산 확인
	ㅇ 메뉴 별 통계 조회 : 직원(고객) -> 시스템 -> 메뉴별 구입 내역(연령 별 조회, 성별 조회)를 조회
		+ 연령 별 조회
		+ 성 별 조회
	ㅇ 문의사항 관리 : 직원(고객) -> 시스템 -> 문의사항
	----- 최고 시스템 관리자 관리자 -----
	ㅇ 시스템 관리자 등록 : 최고시스템 관리자(admin) -> 매장별 관리자 등록
	ㅇ + 시스템 서비스 관리 : 시스템의 서비스를 추가, 삭제, 수정

	----- 결제 시스템 -----
	ㅇ + 결제 내역 등록 : 결제시스템 -> 시스템 : 결제가 완료된 주문 내역을 등록한다
	ㅇ + 적립금 적립 : 결제 시스템 -> 시스템 : 로그인 후 완료된 결제에 대해 적립금 적립
